# user-grouping-kmeans
基于 K-Means 聚类算法，对达人4周收益进行自动化用户分层，并按规则计算活动奖金。

## 计算逻辑
1. 数据清洗：过滤四周收益均极低的无效达人（代码里是 < 10）
2. 特征计算：生成「四周总收益」用于聚类
3. 聚类分层：使用 K-Means 算法自动分为 8 个层级
4. 奖金规则：
   - 总奖金 = 全量达人总收益 × 0.03
   - 层级1不发放奖金
   - 层级2~8按各层级总收益占比分配奖金
   - 同层级内人均奖金相同
5. 输出结果：层级范围、单人奖金、分层结果 CSV 文件

# creator-income-level-bonus
基于已有的层级划分标准,按达人4周收益数据完成用户最终层级判定、奖金池计算。

## 项目功能
- 按固定收益区间自动划分8个用户层级
- 基于4周收益计算每人每周层级与最终层级
- 自动筛选获奖达人（层级≥2）
- 计算总奖金池（获奖总收益 × 3.5%）
- 按层级收益占比分发奖金，同层级人均奖金一致
- 输出各层级内每个达人的奖金

## 层级划分标准（共8级）
1. 0–300
2. 301–1000
3. 1001–3000
4. 3001–5000
5. 5001–8000
6. 8001–15000
7. 15001–30000
8. 30001–70000
---

## 最终层级判定逻辑
1. 统计4周中出现次数最多的层级
2. 若出现次数相同：
   → 计算**对应周收益总和**，取总和更大的层级
3. 无有效数据 → 默认层级1
---

## 奖金计算逻辑
1. 获奖条件：最终层级 ≥ 2
2. 总奖金池 = 所有获奖达人4周总收益 × 0.035
3. 按各层级**总收益占比**分配总奖金
4. 同层级内：人均奖金 = 层级总奖金 ÷ 该层级人数
